# 两个小基准，三个可复现的发现：关于"能跑通的评测"的方法论笔记

> 作者：uninhibited-scholar · 2026-07-11
> 本文所有数字可从两个公开仓库零依赖复现：
> [fraud-detect-bench-zh](https://github.com/uninhibited-scholar/fraud-detect-bench-zh) · [agent-endurance-bench](https://github.com/uninhibited-scholar/agent-endurance-bench)
> 先说清边界：这是**小规模、公开标签、确定性可复现的种子评测**，全部结论为**基准内结论**，不外推模型一般能力或真实部署表现。

## 为什么做小基准

大部分个人做评测会死在两个坑里：要么数据造不动（想一步到位几千条），要么结论撑不住（LLM 裁判、隐藏口径、不可复现）。我选了另一条路：**把"小"当成设计约束而不是缺陷**——样本少，但每条 gold 封闭可重算；规模小，但校验器、manifest 哈希锁、CI 确定性锁让任何改动都进 git diff；模型少（2 个），但预测文件原样入库，任何人可以重跑我的评分器验证每一个数字。

这条路线在一次真实的外部审查中被检验过：审查者独立复现了全部主榜数字，同时指出了三处夸大（"封闭 gold"其实是公开 gold、"CI 防作弊"当时并未真正落地、统计表述过强）。全部整改后复审通过。**这个来回本身就是方法论的一部分**：小基准的可信度不来自规模，来自可核查性。

## 发现一：记得规则，算错账，账越长错得越多

agent-endurance-bench 测的是：多轮累积对话跑到第 50/100/200 步之后，模型还记得开局约束吗？账算得对吗？

glm-5.2 的失效方式非常具体：**规则记忆 96.5%、抗干扰 100%，但累计预算的算术状态随深度漂移**——state 探针准确率从 v1 的 0.812 掉到 v2（含 200 步高压层）的 0.681，101–200 步段只剩 70%（21/30，Wilson 95% CI [0.52,0.83]，样本少、区间宽，方向明确但幅度估计不稳）。错误全部是级联误差（一步算错、后续全错）或答成过期旧值。

也就是说，在这个构造下它不是"忘了你说过什么"，而是"记得规则、算错账"。这两种失效的工程对策完全不同：前者要记忆系统，后者要外置状态（把账本交给工具而不是上下文）。

deepseek-v4-flash 则 180 个探针全对，v1/v2 双轮满分——对它我们只能给"≥此难度无退化"的下界结论。基准饱和也是信息：它直接给出了 v3 的扩难方向。

## 发现二：判"是不是诈骗"已饱和，归因才是战场

fraud-detect-bench-zh 测模型能否读一条中文短信判断是否诈骗、是哪类诈骗。120 条（82 诈骗/38 正规硬负样本）上：

- 关键词护栏（"见到转账/验证码就报警"）：recall 51%，误报 40%，detection_gap 0.117——接近瞎报
- 两个推理模型：**82/82 全部识别**，误报 1/38 和 2/38

verdict 这一层已经被打满，两模型间差距不显著（区间重叠）。但**类别归因**被 v2 的混合话术设计拉开了：28 条"贷款/物流/退款骗局刻意掺机关字样"的样本上，glm 21/28、deepseek 17/28。

## 发现三：机关字样是双向陷阱

错误方向高度一致：骗局里出现"银保监会""禁毒大队""税务稽查"，模型就把贷款诈骗、物流诈骗归成"冒充公检法"——表面词压过了行为结构（先交钱=贷款诈骗，这个结构信号被机关词淹没）。

更有意思的是反向：deepseek 把 2 条**真实的**机关通知（公安反诈提醒、税务退税到账）误报成诈骗。同一批表面词，正反两个方向都能把模型带偏。对做反诈落地的人，这意味着：模型需要的不是"机关词警觉"，而是**行为结构判别**——谁在要钱、要信息、要操作。

## 方法论清单（可直接抄）

1. **gold 必须可重算**：endurance 的预算 gold 由每步花费累加重建，校验器重算核对——改 gold 必被抓。
2. **改动必须可见**：fraud 用 manifest（精确 ID 集 + 逐条 SHA-256），endurance 用 CI 确定性锁（重跑种子化生成器 + `git diff --exit-code`）。这不防蓄意作弊，防的是"悄悄改数据"——所有变更强制进入可审查的 diff。
3. **评分器也会错**：deepseek 曾"错" 11 题，人工核查全是解析缺陷（答对但附说明文字）。修复后满分。**永远人工抽查失败样例**，评分器修复过程写进 README。
4. **统计要克制**：报原始计数 + Wilson 95% CI，明说哪些差距不构成排序。
5. **饱和是信号不是失败**：模型打满 → 下一版扩难方向就有了；基准的生命力在版本迭代。
